1. ля того, чтобы скопировать репозиторий из GitHub на свой компьютер, вам потребуется выполнить несколько простых шагов:

Скопируйте URL-адрес репозитория, который вы хотите скопировать. Для этого перейдите на страницу репозитория на сайте GitHub и найдите зеленую кнопку "Clone or download". Нажмите на неё и скопируйте URL-адрес.

Установите Git на свой компьютер, если его еще нет. Это можно сделать, загрузив установочный файл Git с официального сайта https://git-scm.com/downloads и следуя инструкциям по установке.

Откройте терминал на своем компьютере и перейдите в каталог, в который вы хотите скопировать репозиторий.

В терминале введите команду "git clone", затем вставьте скопированный ранее URL-адрес репозитория и нажмите клавишу Enter.

Пример команды:

git clone https://github.com/username/repository.git

Git скопирует репозиторий на ваш компьютер. Когда процесс закончится, вы сможете работать с репозиторием на своем компьютере.

2.Для того, чтобы получить изменения из удаленного репозитория на GitHub на локальный компьютер, вам нужно выполнить команду git pull. Эта команда загрузит любые новые коммиты из удаленного репозитория и автоматически объединит их с вашей локальной веткой.

Вот пример использования команды git pull:

Откройте терминал на своем компьютере и перейдите в локальный репозиторий, используя команду cd:
cd /path/to/local/repository
Выполните команду git pull, чтобы получить изменения из удаленного репозитория:
git pull
Git загрузит все изменения из удаленного репозитория и автоматически объединит их с вашей локальной веткой.
Обратите внимание, что если вы работаете с ветками, вам нужно будет указать имя ветки при выполнении команды git pull, например:
git pull origin my-branch
Эта команда загрузит изменения из удаленной ветки my-branch в репозитории на GitHub и объединит их с вашей локальной веткой my-branch.

3.Команда git status позволяет узнать текущее состояние вашего рабочего дерева и индекса. Она показывает, какие файлы были изменены, добавлены или удалены, и указывает, какие изменения были проиндексированы и готовы к коммиту. git status также сообщает о том, какая ветка (или ветки) находится в данный момент в рабочем дереве.

Команда git log выводит список коммитов в вашем репозитории. Эта команда показывает историю изменений в вашем проекте и содержит информацию о каждом коммите, такую как дата и время коммита, автор коммита, сообщение коммита и хеш коммита. Команда git log также может использоваться для просмотра истории изменений в отдельной ветке или для просмотра коммитов, сделанных в определенном интервале времени.

Команда git remote -v выводит список всех удаленных репозиториев, связанных с вашим локальным репозиторием. Каждый удаленный репозиторий имеет имя (обычно "origin"), а также URL-адрес, по которому можно получить к нему доступ. Команда git remote -v также показывает, какой из удаленных репозиториев используется для чтения и записи данных (обычно это "origin"), и какой URL-адрес используется для каждого удаленного репозитория.
